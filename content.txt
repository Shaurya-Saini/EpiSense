# Very Short Description (1–2 lines)

An IoT and machine learning system that monitors environmental indicators such as turbidity, TDS, and temperature to detect conditions associated with infectious disease outbreaks. Sensor data is analyzed using ML models and visualized through a real-time dashboard for early risk detection.

---

# Project Overview

Early detection of infectious disease outbreaks remains a significant challenge in many regions due to delayed reporting systems and limited environmental monitoring. Traditional disease surveillance methods rely primarily on laboratory confirmations and hospital reports, which often occur after the disease has already begun spreading within communities. 

Environmental factors such as contaminated water and changing environmental conditions frequently act as early indicators for the spread of infectious diseases. However, these parameters are rarely monitored continuously at the community level. The absence of real-time environmental data limits the ability of health systems to detect early warning signals that could indicate potential outbreaks. 

This project proposes an **IoT-based environmental sensing system integrated with machine learning analytics** to detect patterns associated with potential disease outbreaks. A sensor node built using an ESP32 collects environmental parameters including **Total Dissolved Solids (TDS), turbidity, and water temperature**, which are indicators of water contamination and environmental conditions linked to waterborne diseases.

The collected sensor data is transmitted to a backend server where a **machine learning model trained on historical water quality datasets** analyzes incoming data to estimate potential outbreak risk conditions. The results are visualized on a **real-time web dashboard**, displaying live sensor readings, predicted risk classifications, and historical trends.

By combining **continuous environmental sensing, machine learning prediction, and real-time data visualization**, the system demonstrates how IoT technologies can support **early warning mechanisms for infectious disease surveillance**, enabling faster responses and preventive actions.

Future versions of the system may incorporate **hospital symptom data APIs and epidemiological datasets** to enhance predictive accuracy and provide a more comprehensive outbreak detection platform.

# System Architecture

The system follows a **layered IoT-AI architecture** that separates sensing, processing, and visualization components.

**Environmental Sensing Layer**
Sensors continuously measure environmental indicators associated with water contamination and disease spread, including turbidity, TDS, and temperature.

**Edge Device Layer**
An ESP32 microcontroller collects sensor readings and transmits the data via WiFi to the backend server through REST API requests.

**Backend Processing Layer**
A FastAPI server receives the incoming data and processes it using a trained machine learning model that evaluates environmental conditions and predicts potential disease risk indicators.

**Real-Time Communication Layer**
Prediction results and sensor values are broadcast to connected clients using WebSockets.

**Application Layer**
A React-based web dashboard visualizes live sensor data streams, predicted risk levels, and historical environmental trends.

System flow:

Sensors
↓
ESP32
↓ WiFi
REST API
↓
FastAPI Backend
↓
Machine Learning Model
↓
WebSocket Stream
↓
React Web Dashboard

# Machine Learning Component

The machine learning component analyzes environmental sensor data to identify patterns that may indicate conditions favorable for infectious disease spread.

The model is trained using the **Water Potability Dataset from Kaggle**, which contains physicochemical parameters of water samples and corresponding safety classifications.

Selected features relevant to the implemented sensor hardware include:

* Total Dissolved Solids (TDS)
* Turbidity
* Temperature

During runtime:

1. Sensor readings are transmitted to the backend server.
2. The trained ML model processes the incoming feature values.
3. The model predicts a **risk classification associated with potential disease-related environmental conditions**.
4. The prediction and sensor values are displayed on the dashboard in real time.
